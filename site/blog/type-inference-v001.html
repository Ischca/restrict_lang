Type inference in the v0.0.1 surface

Release note draft

Restrict's current inference work is centered on a constraint-based, bidirectional core. Generic calls, function values, expected lambda propagation, and empty local container inference are part of the release-facing path.

The important boundary is what the release does not promise. Temporal affine types, source-level forms, user-defined ADTs, and host-visible generic or composite exports remain outside v0.0.1. 

fun identity: <T>(value: T) -> T = {
    value
}

export fun answer: () -> Int32 = {
    42 |> identity
}

That gives the compiler room to make everyday generic code ergonomic without treating the host ABI as solved before it is designed.
