Dogfooding Restrict with runtime examples

Release note draft

Type-check-only examples are useful, but they do not prove that the compiler can lower practical programs into executable WebAssembly. Runtime dogfood examples close that gap.

The pattern is simple: keep rich records, lists, options, results, and lambdas inside Restrict, then expose a small scalar wrapper for the host. That stays inside the v0.0.1 ABI while still exercising real programs. 

export fun dogfood_score: () -> Int32 = {
    val report = () build_report
    report.score
}

This also makes the online compiler more honest. The examples linked from docs and blog posts can represent what the release surface actually supports today.
